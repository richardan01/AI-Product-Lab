# 01 — Faithful to source

**Criterion:** Every claim written or changed in memory this run traces to a specific, locatable source in the window — a transcript passage, a commit, a `Decisions/` file, or a `Tasks/` line.

**Why:** A memory system's whole value is that it is true. An unsourced assertion becomes permanent false context loaded into every future session.

**Pass:** Each new/changed memory line can be pointed to a concrete source. Stakeholder facts name the session or file where they surfaced.

**Fail:** A claim sits in memory with no traceable origin in the window; or a date/number is stated more precisely than any source supports.

**⚠ Partial:** The claim is true but was inferred without a direct source — log it as drift risk.

**How to check:** For a sample of new/changed lines, locate the source in the window. Any that can't be located → ❌.
