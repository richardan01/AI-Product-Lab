# Repo Relationship

```text
Product-Management_OS → AIPM-OS → RegEval
```

## What each layer proves

### Product-Management_OS

`Product-Management_OS` proves the reusable framework: a forkable PM operating system for workflows, memory, knowledge management, quality gates, and eval practice.

It answers: **Can the operating model be reused beyond one person's portfolio?**

### AIPM-OS

`AIPM-OS` proves the applied portfolio lab: the framework running against a real AI PM development path, with public artifacts, project logs, quality gates, and eval discipline.

It answers: **Can the operating model produce inspectable proof-of-work over time?**

### RegEval

RegEval proves the flagship measurable project inside the lab: an LLM-as-judge evaluation loop for regulated-domain compliance classification.

It answers: **Can the portfolio claim be tied to a falsifiable metric, repeatable runs, failure analysis, and documented remediation?**

## In one sentence

`Product-Management_OS` is the reusable system; `AIPM-OS` is the applied lab; RegEval is the measurable proof point.
